Language Choice

\setlang{code}[option]
\setlang*{code}[option]

The code is a Western European language code. Novelette is not configured for other world languages, due to layout calculation methods.

Novelette only uses this setting for hyphenation patterns, and (for French) punctuation spacing. It is not a spelling or grammar check. It does not format time or date. So you may write snippets of text in most other languages, if you do not mind them being hyphenated like your setting.

Hyphenation patterns are provided by a TeX package that supports the chosen language. If you do not already have what you need, use the TeX package manager to install hyphen-language where language is the English name for that language, in lowercase (hyphen-french rather than hyphen-francais). Alternatively, it may be the two-letter language code, as shown below, with hyph suffix (French is frhyph).

Available codes:

  • da (Danish)
  • nl (Dutch)
  • is (Icelandic)
  • ga (An Caighdeán Gaelic)
  • nb (Norwegian Bokmål)
  • nn (Norwegian Nynorsk)
  • sv (Swedish)
  • fi (Finnish)
  • pt (Portuguese)
  • es (Spanish)
  • eu (Basque)
  • ca (Catalan)
  • oc (Occitan)
  • fr (French, France)
  • fr-CH (French, Swiss)
  • de (German, Germany/Austria)
  • de-CH (German, Swiss)
  • it (Italian)
  • en (English, British)
  • en-US (English, USA)
  • xx (No hyphenation).

For most languages, there is no difference between the starred and unstarred \setlang commands, and no options. Exceptions:

In German (de), the unstarred \setlang uses contemporary (post-1996) rules. The starred \setlang* uses older (1901) rules.

In French (fr) the unstarred \setlang adds extra space before the colon, semicolon, exclamation, and question marks. It also adds extra space after opening guillemets, and before closing guillemets. Option [-g] omits the guillemet spacing. Swiss French (fr-CH) is the same as French, except there is no extra space before the colon.

For both French and Swiss French, the starred \setlang* does not add any extra space (whether or not option [-g]). This might be useful if you already wrote spaces manually.
