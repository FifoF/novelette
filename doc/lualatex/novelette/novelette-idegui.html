Writing: What to Use

How to write your book? Should you use a word processor? A plain text editor? A TeX IDE (integrated devlopment environment)? Or one of the TeX GUI (graphical user interface) programs?

The answer is NOT "anything". Some choices are better than others.

Word Processors

Your best choice is probably a word processor. Reason: Such programs usually have built-in spelling check, and can subtitute smart quotes (curly instead of straight). If you are a good typist, this will help you write quickly, instead of being bogged down by TeX code. You can add the TeX code later.

What you need to know:

  • If you do write any TeX code, it will be flagged as mis-spelled. So it is better to turn off automatic spelling correction.
  • How you set page size, margins, text size, font name, or anything else, is irrelevant. That information will not be transferred to TeX.
  • Do not use headers or footers. The method used by a word processor is very different from the method used by TeX.
  • TeX separates paragraphs with a blank line. Alternatively, \par ends a paragraph. The word processor may begin a new paragraph whenever you hit Enter, and indent whatever follows. Those indents will be ignored by TeX. The best way to ensure correct paragraphs, is to separate them with a blank line.
  • Text styling, such as Italics, must not be used. You will need to write TeX markup later.
  • Your text must be exported as "plain text" in utf-8 encoding, without BOM (byte order mark). Ensure that it does not add headers or page numbers.
  • If you cannot do all of the above, then do not use a word processor.

Plain Text Editors

A plain text editor is exactly that. What you type is what it saves. Some plain text editors are enhanced with spell check. Some are enhanced with syntax highlighting, which colors the code markup.

Be sure that the plain text editor saves (or exports) its work in utf-8 encoding, without BOM. This is the standard on Linux devices. On Windows, you may need to check the program settings. Under no circumstances use Windows Notepad. The free program Notepad++ is popular for Windows.

Do not enable automatic code completion. Since Novelette has many commands that differ from ordinary LaTeX, the result may be incorrect.

The developer of Novelette uses the "nano" plain text editor, on Linux. Spelling check, via a word processor, is done later. Final TeX markup is in nano or TeXworks (see below).

IDEs and GUIs

There are several programs that provide a graphical interface for TeX documents. Typically, they provide a TeX-oriented menu, a pane or tab where you can write plain text, and another pane or tab where you can view the resulting PDF (after build, not in real time).

However, most of these programs were created for writing mathematical expressions. Their menu choices are irrelevant to popular fiction. Worse, they may offer to insert TeX commands that will fail in Novelette. So, do not use those programs.

The exception is TeXworks. It comes with MiKTeX, and is available on most Linux platforms. Its minimal interface does what you need to do, without offering features that you cannot use. You may be able to add spell-check. If you like TeXworks, use it.

The developer of Novelette is very insistent on having paragraphs, and page breaks, just where they are best-suited to the text. After the writing and markup is tentatively finished, the files are loaded into TeXworks. Where necessary, paragraphs are individually edited. This is the advantage of original fiction; anything can be re-written with more or fewer words.
