Language Choice

Novelette only supports Western European languages (most, not all). This is because many other languages have character shapes that are not consistent with Novelette's layout calculations.

\setlang{code}[option]
\setlang*{code}[option]

For languages other than French or German, there is no difference between the unstarred \setlang and the starred \setlang*. Only French has an option.

Be sure to use the language code, not the language name. Wrong: \setlang{italian}. Correct: \setlang{it}.

If you do not use \setlang, Novelette will use a generic version of English, built into LaTeX. However, if you are writing in English, use either \setlang{en} or \setlang{en-US}

Novelette only uses \setlang for hyphenation patterns, and (for French) punctuation spacing. It is not a spelling or grammar check. It does not format time or date. You may write snippets of text in most other languages, if you do not mind them being hyphenated like your setting.

Hyphenation patterns are provided by a TeX package that supports the chosen language. If you are writing in your own language, the necessary package is probably pre-installed. If you do not already have what you need, use the TeX package manager to install it.

NOTES

1. In German, the unstarred \setlang uses contemporary (post-1996) rules. The starred \setlang* uses older (1901) rules. Swiss German has little or no change from old to new.

2. In German, package dehyph-exptl provides more recent rules than does hyphen-german. Install both packages.

3. French (fr) : The unstarred \setlang adds extra space before the colon, semicolon, exclamation, and question marks. It also adds extra space after opening guillemets, and before closing guillemets. Option [-g] omits the guillemet spacing.

4. Swiss French (fr-CH) : Same as French, except there is no extra space before the colon.

5. Both French and Swiss French : The starred \setlang* does not add any extra space (whether or not option [-g]). This might be useful if you already wrote spaces manually.
