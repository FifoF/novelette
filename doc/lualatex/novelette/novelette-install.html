Installation

Novelette requires a working TeX installation, which does not need to be very large. It requires one or more fonts suitable for printed fiction. It requires utf-8 encoded text.

Basics

^top

When you install TeXlive (or MiKTeX), you need these collections:

  • Essential programs and files
  • LaTeX fundamental packages
  • LuaTeX packages

That is a very small installation, in the TeX world. You will also need these packages, which you may already have:

xparse xpatch xifthen xstring ifmtarg fontspec
fp microtype silence changepage noindentafter

You will need hyphenation patterns for the language of your book (you do not need them for the occasional short phrase in another language). If you did not install the TeXlive collection for your language, you may install only what you need.

In most cases, you need package hyphen-yourlang where yourlang is the English lowercase name of your language (french, not français; italian, not italiano). Disregard regional differences: canadien, suisse, français are all french.

Unless you are an advanced user who writes custom code, you do not need to read the documentation for packages other than Novelette itself. If you are an advanced user, you probably need the documentation for packages fontspec and microtype.

In some Linux operating systems, there is no lualatex binary program. Check: which lualatex. If it is not found, try this: which luahbtex. If that is not found, then your TeX installation is bad. Otherwise, navigate to the directory that has luahbtex and command:

ln -s luahbtex lualatex

Fonts

^top

Do not rely on the default fonts that come with a basic TeX installation. In general, they are not suited to popular fiction.

Novelette can use any OpenType font (extension otf) and some TrueType fonts (extension ttf). It cannot use Type 1 fonts (extension pfb).

The LaTeX Font Catalog shows samples of all fonts that can be installed within TeX. If you like a font, be sure that it is available as OpenType (or TrueType).

You are not limited to LaTeX fonts. On most platforms, Novelette can find fonts installed in the operating system, not just the ones installed by TeX. So, if you have a free/shareware/commercial font that you like, and can install it in your operating system, Novelette can use it.

Tech note for Android/Termux users: You may need to install non-TeX fonts in your texmf-local tree, because you cannot install them in your Android system. Be sure to run mktexlsr afterward.

Font Licenses

^top

Whether you use a LaTeX font, or a font that comes with your operating system, or a font that came with some program, or a font that you installed from any other source: Be sure that the font has a license that permits embedding, for printed books, for profit. This is your responsibility; Novelette cannot check it.

Many "free" or system fonts do not have a suitable license. In particular, DO NOT use any font that comes with the "GNU Public License" (GPL) unless it also has a "font exception" clause that allows embedding for any purpose. Some fonts have a free version, and a paid version. You might need the paid version.

In general, fonts with the LPPL (LaTeX Project Public License), or MIT License, or SIL OFL (Open Font License), can be used.

Font Previews

^top

Your print-on-demand book will be printed using a high-speed photocopier, not offset press (unless you strike it rich). Before committing to a font, create test files, for each Font Name.

Use the following code. Select either letter or A4 depending on what size paper is used. This is not the same as your book page size:

\documentclass{novelette}
\title{Font Test File}
\author{Novelette}
\setfont{main}{Font Name}
\sheetfeed{letter or A4}
\labelfont
\begin{document}
\fillertext
\end{document}

Print the PDF on a photocopier that can print directly from digital files. Where I live, this kind of machine is often at print/ship business services. Be sure to specify non-color print.

Preamble command \labelfont prints the name of the main font in the top left margin of the first page. That way, you can print several test files, and know which font is which.

Pay particular attention to how dark the characters print. Fonts intended for offset presses may print too light. Pay attention to the reading experience: Does the text flow the way you expect in fiction, or does it look more like a textbook?

Only you can decide.

Alternate Font

^top

Some very nice OpenType and TrueType fonts do not have an extended character set. Usually, no problem. If you are writing in one of the major Western European languages, almost any font has what you need.

But what if you need the occasional character that is not in your chosen font? Novelette provides \altfont{character} for this purpose. The character is picked from font Libertinus Serif, scaled to match surrounding lowercase text.

For that to work properly, you must have Libertinus Serif fonts installed where TeX can find them. You only need the fonts, not support for math or anything else. Command line:

tlmgr install libertinus-fonts

Why Libertinus Serif? It has moderate design that will be inconspicuous when you insert the occasional character. It has a large character set, with many OpenType Features. In fact, it is worth consideration as your main font.

If you do not use \altfont, then it does not matter. But if you use \altfont and Libertinus Serif is not installed, then Novelette will issue a warning, use Latin Modern Roman there, and prevent final mode.
