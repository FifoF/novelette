Special Characters (or Not)

TeX is notorious for its numerous characters with special meaning:

\ { } ~ % # $ ^ & _

Some other characters have special meaning, but only in a particular context.

Special Characters in Preamble

In the Preamble, special characters always have their TeX special meanings. This may cause errors. The most likely errors will result if you use a special character in the title or author. Examples of bad code:

\title{The 20% Conspiracy}
\title{Jack & Jill}

You may remove the special meaning with backslash in front of the character. Examples of good code:

\title{The 20\% Conspiracy}
\title{Jack \& Jill}

Special Characters in Document Body

After \begin{document}, Novelette removes the special meaning from these characters:

# $ ^ & _

If you type any of the above characters, they will simply print. It does not matter whether or not you put a backslash in front of them.

Novelette has its own commands for superscript, subscript, and Unicode character code. In most cases, you cannot define macros within the document body. You cannot enter math mode, create tables, or do tab alignment.

These characters retain their special meaning:

\ (backslash) begins a command. If you need to print a backslash character, use \textbackslash instead.

{ and } create a group. If you need to print those characters, use \{ and \} instead.

~ (tilde) is a non-breaking space. If you need to print the tilde character (rather than place a tilde over another character), use \textasciitilde instead.

% (percent) begins a comment. It does not print, and anything after it on the same line (of plain text) does not print. If you wish to print % and also print what follows, use \% instead.

Actually, it is better to not use % anywhere in the document body. Did you really mean comment, or did you really mean to print % there? TeX cannot read your mind, and does not throw an error or warning. But Novelette helps you: If it sees % in document body, without preceding backslash, it writes an ALERT message in the log file. Then you can either change the naked % to \% so that it prints, or use \memo{} if you meant comment.
