Novelette - Documentation

Document class novelette is designed for one purpose only: popular fiction, printed to paper (not Ebooks), black ink, especially using print-on-demand technology. It has what is necessary for that purpose, and no more. Much of the vast LaTeX code is disabled or ignored. The result is a small set of commands that have simple names and simple usage. They do what you need, allowing you to focus on writing your book, instead of manipulating LaTeX code.

Here is a model document template. Each of its commands is a hyperlink within this documentation. Click to learn what the commands do, and how to choose their values:

% !TeX TS-program = lualatex
% !TeX encoding = UTF-8
\documentclass{novelette}
\mode{draft}
\intent{SWOP}
\trimsize{width=5.5in,height=8.5in}
\sheetfeed{trimsize} % Normally not used.
\margins{top=.5in,outside=.5in,bottom=.5in,inside=.625in}
\design{chars=62,lines=30,style=2,headgap=1.5,footgap=1.5}
\setfont{main}{EB Garamond}[liga]
\openings{top=5,height=14,align=center}
\begin{document}
% Your writing goes here.
\end{document}

% !TeX TS-program = lualatex

This is a "magic comment". Some GUI programs (TeXworks, TeXstudio, etc.) can read this line. It tells them to use lualatex compiler, regardless of what the program uses for its standard compiler.

If you are compiling from command line, be sure to use lualatex. Anything else will be rejected.

% !TeX encoding = UTF-8

This is a "magic comment". It tells some GUI programs that the characters have utf-8 encoding. You must type your document using utf-8, without its "byte order mark" (BOM). This is essential. It is the default encoding for Linux and Mac. On Windows, be sure to tell configure text editor for utf-8 without BOM.

\documentclass{novelette}

Unlike other LaTeX document classes, Novelette has no class options. Anything you need is set by one of the other commands in Preamble, or is automatically calculated from other settings.

\mode{choose}

Novelette has four modes: draft, shade, align, final. If you do not set the mode, its default is draft.

draft disables PDF/X conformance (see intent). It places DRAFT at the top left of each page. The PDF title is changed, to include the word DRAFT.

Otherwise, the layout, line breaks, and page breaks are the same as they will be, when you graduate to final mode.

shade is draft mode, with margins tinted gray.

align is draft mode, with margins tinted gray, and with text baselines marked.

final should only be used when you are ready. In final mode, Novelette performs some checks that it would otherwise omit (to save time). If your document fails one of the checks, you will be warned. It will automatically revert to draft mode, so that the PDF cannot be confused with the final file.

\intent{choose}

There are five intents: none, SWOP, FOGRA, JC, custom. The default is none.

No matter which intent you choose, fonts are subset embedded. That is enough for some print services. But many print services require PDF/X conformance, especially PDF/X-1a:2001.

When a PDF file conforms to PDF/X, it must have a certain internal structure. Among the requirements is an "output intent", which states that the PDF was prepared according to the requirements of a particular print technology. It does not force the printer to use that technology. In fact, your file will be printed by entirely different methods. Nevertheless, an intent must be declared, even though it is useless.

You are strongly urged to use an intent, other than none, for the final PDF. This enables additional checks, hopefully reducing the possibility that your file will be rejected by the print service.

none does not produce PDF/X. No matter how you set intent, it will be automatically changed to none unless you are in final mode.

SWOP means "US Web Coated SWOP". This is most frequently used in the United States.

FOGRA means "FOGRA39L". This is frequently used in Europe.

JC means "JC200103". This is frequently used in Japan.

custom means that you will provide the data for some other intent. You must code the data yourself. Do not use custom unless specifically required by your print service.

\trimsize{width=W,height=H}

Trim Size is the size of your book pages. Be sure to choose cream paper (creme, off-white), not white paper. The print service will offer a choice of several standard sizes.

Dimensions W and H are measured in units of inches or millimeters (in,mm).

The name "trim size" comes from the manufacturing process. Softcover books, especially print-on-demand, are usually printed using Perfect Print technology. The pages are printed on larger paper, roughly cut and assembled with glue, the cover is attached with glue, then the entire book is "trimmed" with a cutting device. This is why Perfect Print books have all page edges aligned.

In the case of library hardcover books, trim size refers to the interior pages. The cover size is slightly larger.

\sheetfeed{choose}

Do not use the \sheetfeed setting, unless you are certain that you need it. If you use this setting incorrectly, big problems will result. Available choices are trimsize, letter, A4.

trimsize is the default. The PDF page size will be the same as the book trim size. This is how nearly all print-on-demand services want it.

letter forces the PDF page size to USletter, 8.5in width, 11in height. The book trim size is centered vertically and horizontally.

A4 (or a4) forces the PDF page size to European A4, 210mm width, 297mm height. The book trim size is cenetered vertically and horizontally.

If you are printing pages at home, for manual editing, then you might use letter or A4, to produce a consistent result from your home printer. That might also be the case, if your book will be reviewed by an editor.

Novelette does not produce crop marks or trim marks. It does write the internal "TrimBox" metadata to the PDF.

\margins{top=T,outside=O,bottom=B,inside=I}

In Novelette, margins surround the printable area. Nothing may be printed within the margins. The header, main text, and footer all must fit within the rectangle surrounded by the margins. This is how the term "margins" is used by most print services.

Word processors usually place header within the top margin, and footer within the bottom margin. You cannot do that in Novelette.

Margins are measured in units of inches or millimeters (in,mm). It is best to use the same units for margins, as you use for trim size.

top margin is the area above the topmost printable item, which is the header (if your page style uses header), or main text (if otherwise).

outside margin is the area at the outer page edge, being the left side of even-numbered (verso) pages, or the right side of odd-numbered (recto) pages.

bottom margin is the area below the lowest printable item, which is the footer (if your page layout uses footer), or main text (if otherwise).

inside margin as at the spine (binding). In general, this margin should be set 0.125in (or 3mm) wider than the ouside margin. The extra width allows for binding glue.

You may abbreviate the margins setting, with a single value:
\margins{size} sets top, ouside, and bottom margins to that size. Inside margin is set 0.125in (or 3mm) larger.

If you do not set margins, Novelette will set them for you. At smaller trim sizes, the margins will be at the minimum required by print services. At larger trim size, the margins will be larger.

\design{chars=C,lines=L,style=S,headgap=H,footgap=F}

This setting determines page layout. In Novelette, you do not set either the font size, or the leading (baselineskip). Instead, they are automatically calculated from the trim size, margins, and design.

chars is the typical average number of characters (including spaces) per line of main text. More characters means that more text fits on each line. This is achieved by reducing the font size to fit. The default value for C is 62 for small to medium trim sizes, larger for large trim sizes.

The value is not precise, and it is only an average. It assumes that the font you use, and your style of writing, is "typical" for popular fiction. Do a few test pages, to see what value of C works best.

linesis the ineger number of lines in the main text block. It does not count header or footer. Default value varies, based on other setings.

style is an integer from 1 to 7. It determines whether page layout has header and/or footer, and determines alignment of header content. Other document classes require you to write code for the style. Novelette does that for you.

The default style depends on how much vertical room is available, based on other settings. If enough room, style 2 is default. If not enough room, style 4.

style=1 provides only footer. Page number centered in footer.

style=2 provides both header and footer. Page number centered in footer. Text cenetered in header.

style=3 provides both header and footer. Page number in footer, aligned to outside margin. Text in header aligned to outside margin.

style=4 provides only header. Page number at outside margin. Text cenetered.

style=5 is like 4. Adds a vertical bar near the page number, at fixed distance from the outside margin.

style=6 is like 4. Adds a slash next to the page number.

style=7 provides only header. Page number and text are centered, separated by slash. Page number is toward the outside.

headgap is a number (integer or decimal) from 0 to 1. It defines how much of a "gap" appears between the header and the top of main text. Value 0 is rarely useful. Value 1 skips a whole line, and is best for the large book sizes. Value 0.5 is a nice compromise for smaller books.

footgap works like headgap, except it applies between main text and footer.

If your design has both header and footer, it is best to set headgap and footgap at the same value. But this is not requird.

\setfont{role}{font name}[features]

Novelette uses fontspec technology for Open Type fonts. If you are a knowledgeable LaTeX user, you may use some aspects of fontspec directly. However, do not use \setmainfont.

The \setfont command has two mandatory arguments, and an optional argument.

role is one of these keywords: main head note name desc deco alt. It assigns the font to its proper place and purpose.

font name is the Open Type font family name. Examples are Swainson, EB Garamond, Libertinus Serif. You cannot use TeX fonts (which use three-letter codes).

features is an optional, comma-separated list of Open Type features, using fontspec syntax.

main role is equivlanet to \setmainfont. The font and features will be used for main body text.

head is applied to only the header and footer. In many cases, this is the same as the main font, except do not use ligatures. You might prefer to use Scale=N where N is a number slightly less than 1.

note is used for footnotes and "legalese" text. It is almost always the same as main font, except with Scale less than 1.

name is the font used by the opening \name command. It may be the same as the main font, but at larger Scale. Or you may use a stylish font.

desc is the font used by the opening \desc command. It is usually the same as the main font, but at larger Scale.
