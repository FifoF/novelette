Novelette - Documentation

THIS HTML FILE IS A WORK IN PROGRESS. ANYTHING MAY CHANGE.

Document class novelette is unlike any other LaTeX document class: (a) It requires LuaLaTeX; (b) It does not honor widely-used LaTeX commands; (c) It has its own methods for setting layout and other properties, specifically written for those who have very little knowledge of LaTeX; (d) The resulting PDF is created for printed (to paper) popular fiction, such as detective novels, printed in black ink only; (e) There is no capability for Ebooks or any kind of academic works.

QUICK START: Here is a model document template. Each of its commands is a hyperlink within this documentation. Click to learn what the commands do, and how to choose their values:

% !TeX TS-program = lualatex
% !TeX encoding = UTF-8
\documentclass{novelette}
\mode{draft}
\intent{SWOP}
\setlang{usenglishmax}
\title{Your Wonderful Book}
\author{Famous Author}
\trimsize{width=5.5in,height=8.5in}
\sheetfeed{choice} % Normally not used.
\margins{top=.5in,outside=.5in,bottom=.5in,inside=.625in}
\design{fontsize=11.6pt,lines=30,style=2,headgap=1.5,footgap=1.5}
\setfont{main}{EB Garamond}[liga]
\openings{top=5,height=14,align=center}
\begin{document}
% Your writing goes here.
\end{document}

% !TeX TS-program = lualatex

This is a "magic comment". Some GUI programs (TeXworks, TeXstudio, etc.) can read this line. It tells them to use lualatex compiler, regardless of what the program uses for its standard compiler.

If you are compiling from command line, be sure to use lualatex. Anything else will be rejected.

% !TeX encoding = UTF-8

This is a "magic comment". It tells some GUI programs that the characters have utf-8 encoding. You must type your document using utf-8, without its "byte order mark" (BOM). This is essential. It is the default encoding for Linux and Mac. On Windows, be sure to configure your text editor for utf-8 without BOM.

\documentclass{novelette}

Unlike other LaTeX document classes, Novelette has no class options. Anything you need is set by one of the other commands in Preamble, or is automatically calculated from other settings.

\mode{choose}

Novelette has four modes: draft, shade, align, final. If you do not set the mode, its default is draft.

draft disables PDF/X conformance (see intent). It places DRAFT at the top left of each page. The PDF title is changed, to include the word DRAFT. Otherwise, the layout, line breaks, and page breaks are the same as they will be, when you graduate to final mode.

shade is draft mode, with margins tinted gray.

align is shade/draft mode, with text baselines marked.

final should only be used when you are ready. In final mode, Novelette performs some checks that it would otherwise omit (to save time). If your document fails one of the checks, you will be warned. Then itt will automatically revert to draft mode, so that the PDF cannot be confused with the final file.

\intent{choose}

There are four intents: none, SWOP, FOGRA39, JC200103. Default is none.

No matter which intent you choose, fonts are subset embedded. That is enough for some print services. But many print services require PDF/X conformance, especially PDF/X-1a:2001.

When a file conforms to PDF/X, it must have a certain internal structure. Among the requirements is an "output intent", which states that the PDF was prepared according to the requirements of a particular print technology. It does not force the printer to use that technology.

Your file will be printed by an industrial-strength photocopier or inkjet, rather than the offset presses declared by intent. In any case, the intent is irrelevant for books using only black ink. Nevertheless, an intent must be declared, even though it is useless; it is part of the PDF/X standards.

You are urged to use an intent, other than none, for the final PDF. This enables additional checks, hopefully reducing the possibility that your file will be rejected by the print service.

none does not produce PDF/X. No matter how you set intent, it will automatically be changed to none unless you are in final mode.

SWOP means "US Web Coated SWOP". This is most frequently used in the United States.

FOGRA39 (actually FOGRA39L) is frequently used in Europe.

JC200103 is frequently used in Japan.

\setlang{code}

This command sets the language hyphenation pattern. Only one language may be chosen, from a list. It does not set anything other than hyphenation.

You may write some text in another language; but if you do, it will be hyphenated according to the rules chosen by \setlang.

\title{Title of Your Book}

When you use a print-on-demand service, you will provide the book title. This \title must be exactly the same. Also, it must be the same on the printed title page, on the copyright page, in page headers (if you use them), and generally anywhere the title is expected. Novelette does not do cover artwork, but the cover must have exactly the same title.

The title must use the Latin character sets. Some print services limit to Latin-1; others allow more extended Latin characters. Curly apostrophe and curly quotes are not Latin characters, but Novelette will convert them to the acceptable straight variety.

\author{Author of Your Book}

The considerations for \title apply, with these additional:

Novelette is not intended for academic works. There should be a single author (in some cases, two equal authors, using a single \author).

Whether the author is your real name, or a pseudonym, is up to you. If in doubt, ask the print service.

If you use a pseudonym, then you should probably use your real name as copyright holder, on the copyright page. Then, if someone unlawfully copies your book, it will be easier to enforce the copyright. Otherwise, you will have to prove that "Fakename Author" is actually "Realname You" before distributors will honor your requests.

And, if you do use a pseudonym, request that distributors (and libraries) have it indexed under the desired name (pseudonym or real name).

\trimsize{width=W,height=H}

Trim Size is the size of your book pages. Be sure to choose cream paper (creme, off-white), not white paper. The print service will offer a choice of several standard sizes.

Dimensions W and H are measured in units of inches or millimeters (in,mm). Examples:

\trimsize{5.5in,8.5in} % American Digest.

\trimsize{width=129mm,height=198mm} % British B-Format.

The name "trim size" comes from the manufacturing process. The pages are printed on larger paper, then roughly cut and assembled with glue. The cover is attached with glue, then the entire book is "trimmed" with a cutting device. This is why such books have all page edges aligned.

In the case of library hardcover books, trim size refers to the interior pages. The cover size is slightly larger.

\sheetfeed{choice}

Do not use the \sheetfeed setting, unless you are certain that you need it. If you use this setting incorrectly, big problems will result. Do not guess. Available choices are letter and A4.

If you do not use \sheetfeed, the PDF page size will be the same as the book trim size. This is how nearly all print-on-demand services want it.

letter centers the trim size, vertically and horizontally, in the larger USletter size (8.5in width, 11in height),

A4 (or a4) centers the trim size, vertically and horizontally, in the larger European A4 size (210mm width, 297mm height).

If you are printing pages at home, for manual editing, then you might use letter or A4, to produce a consistent result from your home printer. That might also be the case, if your book will be reviewed by an editor.

Novelette does not produce crop marks or trim marks. It does write the internal "TrimBox" metadata to the PDF.

\margins{top=T,outside=O,bottom=B,inside=I}

In Novelette, margins surround the printable area. Nothing may be printed within the margins. The header, main text, and footer all must fit within the rectangle surrounded by the margins. This is how the term "margins" is used by most print services.

Word processors usually place header within the top margin, and footer within the bottom margin.

Margins are measured in units of inches or millimeters (in,mm). It is best to use the same units for margins, as you use for trim size.

top margin is the area above the topmost printable item, which is the header (if your page style uses header), or main text (if otherwise).

outside margin is the area at the outer page edge, being the left side of even-numbered (verso) pages, or the right side of odd-numbered (recto) pages.

bottom margin is the area below the lowest printable item, which is the footer (if your page layout uses footer), or main text (if otherwise).

inside margin as at the spine (binding). In general, this margin should be set 0.125in (or 3mm) wider than the ouside margin. The extra width allows for binding glue.

You may abbreviate the margins setting, with a single value:
\margins{size} sets top, ouside, and bottom margins to that size. Inside margin is set 0.125in (or 3mm) larger.

If you do not set margins, Novelette will set them for you. At smaller trim sizes, the margins will be at the minimum required by print services. At larger trim size, the margins will be larger.

\design{fontsize=M,lines=L,style=S,headgap=H,footgap=F}

This setting determines page layout. In Novelette, you do not set either the font size, or the leading (baselineskip). Instead, they are automatically calculated from the trim size, margins, and design.

fontsize is the size of main body text, with units pt (TeX points) or bp (PostScript points). The value may be integer or decimal. Typical value is 11.6pt. Run a test document and look at the log Summary. It will tell you the estimated average characters per line. Best readability is somewhere near 65 characters. This number is inversely proportional to font size.

linesis the integer number of lines in the main text block. It does not count header or footer. Default value varies, based on other setings.

style is an integer from 1 to 7. It determines whether page layout has header and/or footer, and determines alignment of header content. Other document classes require you to write code for the style. Novelette does that for you.

The default style depends on how much vertical room is available, based on other settings. If enough room, default is 2. If not enough room, 4.

style=1 provides only footer. Page number centered in footer.

style=2 provides header and footer. Page number centered in footer. Text centered in header.

style=3 provides header and footer. Page number in footer, aligned to outside margin. Text in header, aligned to outside margin.

style=4 provides only header. Page number at outside margin. Text centered.

style=5 is like 4. Adds a vertical bar near the page number, at fixed distance from the outside margin.

style=6 is like 4. Adds a slash next to the page number.

style=7 provides only header. Page number and text are centered, separated by slash. Page number is toward the outside.

headgap is a number (integer or decimal) from 0 to 1. It defines how much of a "gap" appears between the header and the top of main text. Value 0 is rarely useful. Value 1 skips a whole line, and is best for the large book sizes. Value 0.5 is a nice compromise for smaller books.

footgap works like headgap, except it applies between main text and footer.

If your design has both header and footer, it is best to set headgap and footgap at the same value. But this is not required.

\setfont{role}{font name}[features]

Novelette uses fontspec technology for Open Type fonts. If you are a knowledgeable LaTeX user, you may use some aspects of fontspec directly. However, do not use \setmainfont.

\setfont has two mandatory arguments, and an optional argument.

role is a keyword: main head note name desc deco alt. It assigns the font to its proper place and purpose.

font name is the Open Type font family name. Examples are Swainson, EB Garamond, Libertinus Serif. No TeX fonts with three-letter codes.

features is an optional, comma-separated list of Open Type features, using fontspec syntax.

main role is equivlanet to \setmainfont. The font and features will be used for main body text.

head is applied to only the header and footer. In many cases, this is the same as the main font, except do not use ligatures. You might prefer to use Scale=N where N is a number slightly less than 1.

note is used for footnotes. It is almost always the same as main font, except with Scale less than 1.

name is the font used by the \name command, in the opening environment. It may be the same as the main font, but at larger Scale. Or you may use a stylish font.

desc is the font used by the \desc command, in the opening environment. It is usually the same as the main font, but at larger Scale.

deco is a font providing unusual characters such as dingbats, flourishes, or unusually-shaped characters.

alt is a font providing characters that cannot be found in the main text font.

\openings{top=T,height=H,align=A}

The opening environment begins such things as chapters. The \openings comand defines the style.

top T is an integer, number of lines from top of ordinary main text block. It is the baseline position of the highest text in an opening.

align A is a keyword: left, right, center, inside, outside. It sets the default alignment of items in an opening

Note that top and height are not enforced. All these do is place markers in the side margins, at the expected vertical positions. Whether or not these markers are honored, is up to you.

\begin{opening} ... \end{opening}

The opening environment is used at the start of new chapters, and canalos be used at the start of chapter-like portions of your book. It does not matter whether the portion is just one page.

Within an opening, you may use some commands that are not available in main text. These are \name and desc (description). In essence, those are chapter title and subtitle (if any).

The \openings command determines the default alignment of items within an opening. It also places markers. To vertically position items, use the \null command, as often as necessary, to insert blank lines.
